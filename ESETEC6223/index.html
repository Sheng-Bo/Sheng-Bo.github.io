1. Introduction

MIT Scratch is a easy-to-use authoring tool for creating multimedia projects and was developed by the Lifelong Kindergarten group at MIT Media Lab led by Mitchel Resnick. It uses colorful blocks to represent programming syntaxes. The aims of it are not only to teach kindergarten kids how to code, but cultivate creative thinking and problem-solving skills. 

The purpose of my project is mainly to review the history of educational programming languages for kids. A statement about my position that whether or not kids should be taught how to code will be made too.

2. Why Should Kids be Taught How to Code?

Mitch Resnick argued that:

  1. coding is not necessary to be “coding” literally.
  2. calling young people the “digital natives” is skeptic because although they have experiences with new technologies, they are still not familiar with using the technologies to create things. In one word, you can read does not mean you are capable of being a writer. Therefore, He advocates that kids should be taught how to code.

3. Issues for Beginners

  1. Interfaces of IDE (Integrated development environment ) are Complicated.
  2. Non graphical approaches are not intuitive. (text-based, flowcharts…)
  3. Basic English proficiencies are needed for non-native speakers, especially east Asian language speakers (Chinese, Japanese, and Korean).
